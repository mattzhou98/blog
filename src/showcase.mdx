# Lab — everything you can put in a post

This page is a living reference for **everything available in a blog post** (and
anywhere else on the site). Each item shows the **source** you'd write, then the
**live result** right below it. Posts live in `src/content/blog/` as `.md` (prose)
or `.mdx` (prose **plus** interactive components).

<div class="callout">
  <div class="ct">heads up — import paths</div>
  Component import paths below are written as they'd appear in a post at
  <code>src/content/blog/your-post.mdx</code> (two folders deep, so
  <code>../../components/…</code>). Only <code>.mdx</code> files can import
  components; plain <code>.md</code> is prose-only.
</div>

## Prose basics (Markdown)

Plain Markdown gives you headings, **bold**, _italic_, `inline code`,
[links](https://astro.build), lists, quotes, rules, and tables.

````md
## A heading

**Bold**, _italic_, `inline code`, and a [link](https://example.com).

- a bullet
- another, with a nested item
  - nested

1. first
2. second

> A blockquote.

| metric | value |
| ------ | ----- |
| posts  | 1     |
````

Renders as:

- a bullet
- another, with a nested item
  - nested

> A blockquote.

| metric | value |
| ------ | ----- |
| posts  | 1     |

## Cross-references (figures, tables, equations)

A custom plugin gives Quarto-style **numbered** figures/tables/equations with
auto-resolving `@`-references and a hover preview. Label a block with
`:::fig{#fig-x}` / `:::tbl{#tbl-x}` / `:::eqn{#eq-x}`, then reference it with
`@fig-x` / `@tbl-x` / `@eq-x` (note equations use the `eq-` prefix).

### Figure

For charts/diagrams whose ink color can't read on both backgrounds, `<ThemeImg>`
swaps a light and a dark variant by theme. (For an image that already works on
both — a photo, say — a plain `![alt](./img.png)` is fine.)

````mdx
import ThemeImg from '../../components/ThemeImg.astro';
import { u } from '../../lib/paths';

:::fig{#fig-views}
<ThemeImg light={u('figs/views-light.svg')} dark={u('figs/views-dark.svg')} alt="weekly views" />

Weekly views over the first 12 weeks.
:::

As @fig-views shows, growth compounds.
````

Renders as (flip the theme toggle — the figure swaps; hover the reference for a preview):

:::fig{#fig-views}
<ThemeImg light={u('figs/exec-views-light.svg')} dark={u('figs/exec-views-dark.svg')} alt="weekly views, first 12 weeks" />

Weekly views over the first 12 weeks (generated at build by `make_figs.py`, one variant per theme).
:::

As @fig-views shows, growth compounds.

## Runnable code (Python in the browser)

`RunPython` is a single editable, runnable cell. Python runs in a Web Worker via
WebAssembly (Pyodide) — off the main thread, so a runaway loop can't freeze the
page. Pass `code` to seed it.

````mdx
import RunPython from '../../components/RunPython.tsx';

<RunPython client:visible code={`for i in range(5):
    print('row', i, '->', i * i)`} />
````

Renders as:

<RunPython client:visible code={`for i in range(5):
    print('row', i, '->', i * i)`} />

Renders as:

### Scientific notebook (numpy / pandas / matplotlib)

Add `scientific` and each cell lazily downloads only the packages it imports;
DataFrames render as HTML tables and matplotlib figures as inline SVG. (First run
downloads the stack, then it's cached.)

````mdx
<PyNotebook client:visible scientific cells={[

np.random.seed(0)
np.random.randn(5).round(3)`,

pd.DataFrame({"week": [1, 2, 3], "views": [120, 180, 240]})`,
`import matplotlib.pyplot as plt
plt.plot([1, 2, 3, 4], [1, 4, 9, 16], marker="o")
plt.title("squares")`
]} />
````

## Hydration directives

Every interactive component needs a `client:*` directive saying *when* its
JavaScript loads. Everything without one ships as static HTML (zero JS).

| directive | when it hydrates |
| --------- | ---------------- |
| `client:load` | immediately on page load |
| `client:idle` | when the browser goes idle |
| `client:visible` | when it scrolls into view — the best default |
| `client:only="react"` | skip server render; render only in the browser |

That's the whole toolkit. Copy any block above into a `.mdx` post and adjust.
